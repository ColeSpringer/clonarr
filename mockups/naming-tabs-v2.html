Why use a naming scheme?
Adding non-recoverable information (quality, release group, edition, repack status) to filenames prevents duplicate downloads, stops download loops on re-import, and ensures Radarr always knows what you have. Media servers display titles — not filenames — so longer names don't affect your viewing experience.
ⓘ
IMDb vs TMDb
Both IDs are unique. TMDb can occasionally remove IDs and re-add with a new one, but is more aligned with Radarr since it uses TMDb as metadata source. IMDb IDs, once present, are very accurate and rarely change. TMDb is usually better as it guarantees a match — IMDb only gets matched if the TMDb entry has the correct IMDb ID association.

Standard Movie Format
The recommended naming scheme. Includes all non-recoverable information in brackets.
Copy
{Movie CleanTitle} {(Release Year)} - {{Edition Tags}} {[MediaInfo 3D]}{[Custom Formats]}{[Quality Full]}{[MediaInfo VideoDynamicRangeType]}{[Mediainfo AudioCodec}{ Mediainfo AudioChannels]}{[MediaInfo VideoCodec]}-{Release Group}
Movie: The Movie Title (2010) {edition-Ultimate Extended Edition} [IMAX HYBRID][Bluray-1080p Proper][3D][DV HDR10][DTS 5.1][x264]-RlsGrp
Original Title
Uses the release title as-is, which includes all the information from the release itself. Prevents download loops from title/content mismatches, but gives less control over naming. We suggest using {Original Title} instead of {Original Filename} — the filename can be obscured or unclear, especially on Usenet.
Copy
{Original Title}
Movie: The.Movie.Title.2010.REMASTERED.1080p.BluRay.x264-RlsGrp
P2P / Scene Naming
Dots instead of spaces/brackets. Closest to P2P/scene naming, but uses exact audio and HDR formats from the media file.
Copy
{Movie.CleanTitle}.{Release.Year}.{Edition.Tags}.{MediaInfo.3D}.{Custom.Formats}.{Quality.Full}.{MediaInfo.VideoDynamicRangeType}.{MediaInfo.AudioCodec}.{MediaInfo.AudioChannels}.{MediaInfo.VideoCodec}-{Release.Group}
Movie: The.Movie.Title.2010.Ultimate.Extended.Edition.3D.Hybrid.Remux-2160p.TrueHD.Atmos.7.1.DV.HDR10Plus.HEVC-RlsGrp
This naming scheme is designed to work with the New Plex Agent.
⚠
Plex uses {edition-{Edition Tags}} to recognize editions (Director's Cut, Theatrical, etc.) and display them in the UI.
However, if you have two copies with different editions in a merged library (e.g. 1080p Director's Cut + 2160p Theatrical), they will appear as two separate items in Plex.
If you want a movie to appear only once per library, replace {edition-{Edition Tags}} with {Edition Tags}.
